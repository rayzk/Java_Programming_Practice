1.1 The progress of abstraction

All programming language provides abstractions. It can be argued that the complexity of the problems you are able to solve is directly related to the kind and quality of abstraction

Example of steps of abstraction

  • Machine language
  • Assembly language
  • Imperative language (FORTRAN ,BASIC, C)

Approaches of problem solving

  • Imperative language still requires the association between structure of computer and structure of problem (machine model and problem model)
  • Constraint-based programming approach requires to modeling the problem only. LISP and APL choose particular views of the world. (e.g. all problems are lists/decisions)
  • Object-oriented programming approach build connections between elements in the problem space and representations in the solutions space as objects

1.2 An object has an interface

  • Abstract data types are fundamental concept in object-oriented programming, the same as build-in types
  • Interface determines the requests that you can make for a particular object, along with the hidden data comprise the implementation of class

1.3 An object provides services

The program provide service to the user and it accomplish this by using the service offered by other objects. OOP is to produce (or locate in existing code library) a set of objects that provide the ideal service to solve your program

Treating objects as service provider is a great simplifying tool. This is useful not only during the design process, but also when someone is reading your code or reuse the objects.

1.4 The hidden implementation

  • class creator : those who create new data types and build a class that exposes only what is necessary to the client programmer and keeps other things hidden
  • client programmer : the class consumers who use the data types in their applications

The hidden portion usually represents the tender insides of an object that could easily be corrupted by a careless or uniformed client programmer. Hiding the implementation reduces program bugs

With access control, the library designer is able to change the internal working of the classes without worrying about the affects on the client programmer

Access control key words in Java

  • public : the following element is available to everyone
  • private : only available to creator, access a private member will get a compile-time error
  • protected : inheriting class has access to protected members
  • default : (package access) class can access members of other classes in the same package (library component); appears to be private outside the package
